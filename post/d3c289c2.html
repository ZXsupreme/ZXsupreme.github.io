自动化发布机器人部署指南

发表于2026-04-30|更新于2026-04-30|博客
|字数总计:3.4k|阅读时长:15分钟|阅读量:

通过本指南，你可以在无需公网IP的情况下，让飞书群聊中的机器人接收指令（如 /deploy dev comc），自动触发 Jenkins 构建，并在构建完成后返回 Git 提交记录或错误日志。

一、准备工作

项目 要求
一台 Mac 用于长期运行机器人程序（也可用其他电脑，但本文以 Mac 为例）
飞书企业账号 拥有创建“企业自建应用”的权限
Jenkins 服务器 地址、用户名、密码（或 API Token）
Jenkins 任务 已创建好需要触发的任务（如 dev-comc、uat-full-ext），并为每个任务配置相同的远程触发 Token（见下文）

二、飞书应用配置

2.1 创建自建应用

  1. 访问 飞书开放平台 → 点击「创建企业自建应用」。
  2. 填写应用名称（例如“发布机器人”），上传图标，点击「创建」。

2.2 添加机器人能力

  • 进入应用详情页 → 左侧「添加应用能力」→「机器人」→「添加」。

2.3 获取应用凭证

  • 点击左侧「凭证与基础信息」，记下 App ID 和 App Secret（后面会用到）。

2.4 配置权限

  • 点击左侧「权限管理」→「添加权限」，搜索并添加以下3个权限：
    • im:message.p2p_msg （接收单聊消息）
    • im:message.group_at_msg （接收群聊@消息）
    • im:message:send_as_bot （发送消息）
  • 添加后确保状态为“已开通”。

2.5 配置事件订阅（长连接）

  • 点击左侧「事件订阅」→ 在「订阅方式」中选择“使用长连接接收事件”。
  • 点击「添加事件」→ 搜索 im.message.receive_v1 → 勾选并确认。
  • 点击页面底部的「保存」。

2.6 发布应用

  • 点击左侧「版本管理与发布」→「创建版本」。
  • 版本号填写 1.0.0，更新说明任意。
  • 可用范围建议选择“所有员工”（或指定群组）。
  • 点击「保存」→「申请发布」（企业自建应用通常自动审核通过）。

2.7 将机器人加入群聊

  • 打开飞书客户端，进入目标群聊 → 点击右上角「群设置」→「群机器人」→「添加机器人」→ 搜索你的机器人名称并添加。

三、Jenkins 配置（每个任务都需要）

3.1 设置远程触发 Token

  1. 进入 Jenkins 任务配置页面（例如 dev-comc）。
  2. 找到「构建触发器」→ 勾选 「触发远程构建」。
  3. 在「身份验证令牌」中输入一个统一 Token（例如 MyCommonToken2026）。
  4. 点击「保存」。
  5. 所有需要触发的任务都使用同一个 Token。

六、获取群聊 chat_id（用于通知）

  3. 复制该字符串，粘贴到 config.py 的 TARGET_CHAT_ID = "oc_xxxxxxxxxxxx" 中。

九、常见问题与解决

问题 解决方法
机器人无响应 ① 检查飞书后台事件订阅是否已选择“长连接”；② 确认应用已发布；③ 查看终端是否有报错
触发 Jenkins 返回 403 代码已自动获取 crumb，如仍失败，请在 Jenkins 全局安全中临时禁用 CSRF 测试
构建后未收到通知 检查 send_notify.py 路径和权限；确认 TARGET_CHAT_ID 正确；手动运行脚本测试
获取日志提示“无法获取” 确认 Jenkins 任务名包含文件夹路径（如 DICT/dev-comc-ext），且 JENKINS_JOB_PREFIX 配置正确
ModuleNotFoundError: No module named 'lark_oapi' 运行 pip3 install lark-oapi

十、总结

通过以上步骤，你已经拥有一个完全可用的自动化发布机器人，无需公网 IP，仅需一台能联网的 Mac 即可长期运行。团队成员只需在群里 @机器人 + 指令 即可完成部署，构建结果自动反馈到群内。

支持的指令：

  • @机器人 /deploy <环境> <模块名> 例如 @机器人 /deploy dev comc
  • @机器人 /full <环境> 例如 @机器人 /full uat
  • @机器人 /help 查看帮助

可用环境：dev、uat、prod

可用模块：见 config.py 中的 MODULE_SUFFIX_MAP 定义。

如有问题，可以根据终端日志排查，或检查飞书开放平台中的应用配置。
